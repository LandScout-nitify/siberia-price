# Состояние проекта

## Завершённые этапы

- подготовлен устанавливаемый стартовый каркас проекта;
- реализован объяснимый словарный классификатор ВРИ со статусами
  `included`, `excluded` и `manual_review`;
- реализован пилотный CSV → processed Parquet пайплайн с проверкой площади
  и кадастровой стоимости;
- подготовлен raw-слой: атомарное сохранение JSON, устойчивый локальный
  парсер вариантов полей НСПД и синтетические примеры;
- сетевые запросы в тестах не выполняются.

## Текущий этап

Подготовлена инфраструктура пилота по 10–20 вручную заданным кадастровым
номерам Кызыла и пригородных индустриальных территорий:

- tracked-шаблон списка содержит только placeholders;
- реальные номера и raw JSON должны храниться локально в `data/raw/`;
- папка raw JSON обрабатывается в единую проверяемую Parquet-таблицу;
- каждая строка трассируется до имени исходного raw-файла.

## Следующее действие пользователя

1. Сформировать локальный список из 10–20 кадастровых номеров.
2. Вручную сохранить по одной исходной JSON-карточке на номер.
3. Запустить обработку raw-папки.
4. Сопоставить processed-поля с каждой исходной карточкой.
5. Зафиксировать несовпадения полей, спорные ВРИ и пропуски НСПД.

Инструкция приведена в `docs/kyzyl_pilot.md`.

## Ограничения этапа

- сетевой клиент НСПД не реализован;
- автоматический обход карты, кварталов и регионов запрещён;
- реальные кадастровые номера и raw-ответы не публикуются в репозитории;
- пилотная выборка не используется для итогового вывода о завышении;
- до завершения ручного пилота массовый сбор НСПД не запускается.
