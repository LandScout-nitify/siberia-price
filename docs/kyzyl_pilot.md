# Пилот по кадастровым участкам Кызыла

## Цель

Пилот проверяет полную техническую цепочку на 10–20 вручную выбранных
карточках промышленных земель Кызыла и пригородных индустриальных
территорий:

```text
карточка НСПД → raw JSON → parser → проверка качества и ВРИ → Parquet
```

Это проверка качества источника и инфраструктуры, а не статистическое
исследование уровня кадастровой стоимости.

## Почему выборка малая

Малая выборка позволяет вручную сопоставить каждое поле processed-записи с
исходной карточкой, описать варианты структуры ответа НСПД и исправить
парсер до масштабирования. Автоматический обход кварталов, карты или
региона на этом этапе запрещён.

## Список кадастровых номеров

Tracked-шаблон:

```text
data/examples/kyzyl_pilot_cadastral_numbers.csv
```

Он содержит только placeholders и не должен заполняться реальными номерами
в Git. Перед пилотом скопируйте его в игнорируемый raw-каталог:

```powershell
New-Item -ItemType Directory -Force data/raw/nspd/kyzyl_pilot
Copy-Item data/examples/kyzyl_pilot_cadastral_numbers.csv `
  data/raw/nspd/kyzyl_pilot/cadastral_numbers.csv
```

В локальной копии вручную замените placeholders на 10–20 выбранных номеров.
Список используется как контрольный журнал, а не как вход для
автоматического сетевого клиента.

## Сохранение raw JSON

Для каждого номера вручную откройте карточку в разрешённом пользовательском
интерфейсе и сохраните исходный JSON без редактирования в:

```text
data/raw/nspd/kyzyl_pilot/
```

Одна карточка должна соответствовать одному `.json`-файлу. Имена файлов
должны быть уникальными и безопасными для файловой системы, например
`pilot_001.json`. Не включайте в файл cookie, токены, заголовки авторизации
или другие секреты. Raw-файлы игнорируются Git и не должны публиковаться.

Если ответ уже получен разрешённым способом в Python, для его сохранения
используйте `save_raw_json(payload, output_path)`. Функция не выполняет
сетевых запросов.

## Обработка raw-папки

Из корня установленного в editable mode проекта:

```bash
python -m siberia_price.pipeline.process_nspd_raw_folder \
  --input-dir data/raw/nspd/kyzyl_pilot \
  --output data/processed/kyzyl_pilot_processed.parquet
```

Модуль читает только локальные `.json`-файлы верхнего уровня, применяет
существующий парсер, валидацию числовых полей и классификатор ВРИ. Имя
исходного JSON сохраняется в поле `raw_file`. Геометрия сохраняется в
Parquet как JSON-строка.

## Ручные проверки

Для каждой processed-строки сопоставьте с исходной карточкой:

- кадастровый номер;
- площадь;
- кадастровую стоимость;
- категорию земель;
- исходный ВРИ;
- адрес и геометрию;
- рассчитанную кадастровую стоимость 1 м²;
- статус `included`, `excluded` или `manual_review`;
- найденное ключевое слово и причину исключения;
- `data_quality_status`, `data_quality_reason` и `raw_file`.

Все расхождения вариантов полей НСПД фиксируйте отдельно до изменения
парсера. Не исправляйте raw JSON вручную.

## Ограничение выводов

Выборка из 10–20 вручную отобранных участков не является случайной,
репрезентативной или достаточной для оценки регионального уровня стоимости.
По результатам пилота нельзя делать итоговый вывод о завышении кадастровой
стоимости. Допустимые результаты этапа — перечень проблем данных, оценка
полноты полей и решение о готовности инфраструктуры к следующему этапу.

## Ключевой запрет

До завершения пилота по вручную заданным raw JSON не запускать массовый
сбор НСПД.
