# Источники данных

## НСПД

Планируется использовать для получения карточек земельных участков после
отдельной фиксации актуального endpoint, лимитов и правовых ограничений.
На текущем этапе сетевой клиент и массовый сбор не реализованы.

Целевые поля:

- кадастровый номер;
- площадь;
- кадастровая стоимость;
- категория земель;
- ВРИ;
- адрес;
- геометрия;
- статус объекта.

Каждый исходный ответ НСПД должен сохраняться без изменений в raw-слое до
парсинга, нормализации или очистки. Имя файла должно позволять связать raw
JSON с обработанной записью, но не должно содержать секреты или параметры
доступа. Повторная обработка должна читать локально сохранённый JSON, а не
выполнять новый сетевой запрос.

Итоговые расчёты должны быть воспроизводимы по цепочке:

```text
raw JSON НСПД → плоская запись → нормализация и валидация → processed Parquet
```

Синтетические примеры в `data/examples/nspd/` предназначены только для
разработки и тестов. Идентификаторы `SYNTH-NSPD-*` не являются кадастровыми
номерами.

## Фонд данных государственной кадастровой оценки

Используется для проверки:

- отчетов ГКО;
- оценочных групп;
- ценообразующих факторов;
- даты определения кадастровой стоимости;
- методики.

## Росреестр / ЕГРН

Используется для точечной проверки и рыночной верификации.

## Рыночные данные

- сделки ЕГРН;
- объявления;
- банкротные торги;
- инвестиционные площадки;
- муниципальные предложения.

## Экономические данные

- ВРП;
- инвестиции в основной капитал;
- промышленное производство;
- численность населения;
- средняя зарплата;
- логистика;
- индустриальные парки.
