# Методология исследования

## Единица анализа

Один земельный участок.

## Основной показатель

```text
КС 1 м² = кадастровая стоимость участка / площадь участка
```

## Средневзвешенная кадастровая стоимость

```text
Средневзвешенная КС 1 м² = сумма кадастровой стоимости участков / сумма площади участков
```

## Обязательные метрики

- количество участков;
- суммарная площадь;
- средняя КС 1 м²;
- медианная КС 1 м²;
- средневзвешенная КС 1 м²;
- P25;
- P75;
- минимум и максимум после очистки;
- коэффициент вариации;
- индекс Тувы к региону-бенчмарку.

## Локационные классы

| Тип | Описание |
|---|---|
| A | городская промзона |
| B | пригородная индустриальная зона 0–15 км |
| C | периферийная производственная территория 15–50 км |
| D | малый промышленный населенный пункт |
| E | изолированная территория |

## Очистка данных

Исключаются:

- участки без площади;
- участки с нулевой площадью;
- участки без кадастровой стоимости;
- участки с нулевой кадастровой стоимостью;
- непроизводственные ВРИ;
- линейные объекты;
- дороги;
- гигантские участки недропользования — выносятся отдельно.

## Классификация ВРИ

Классификация выполняется детерминированно по словарю
`config/vri_classifier.yaml`. Текст приводится к нижнему регистру, буква
`ё` заменяется на `е`, повторяющиеся пробелы удаляются. Решение всегда
сопровождается найденным ключевым словом, если совпадение существует.

Правила применяются в следующем порядке:

1. При наличии исключающего признака присваивается статус `excluded`.
2. Явно неопределённые и иные формулировки получают `manual_review`.
3. Совпадение ровно с одной группой получает статус `included`.
4. Совпадение с несколькими группами, неизвестный или пустой ВРИ получает
   `manual_review`.

Исключающие признаки имеют приоритет над включающими. Это консервативное
правило не позволяет включить дорогу, линейный объект, жилой или
сельскохозяйственный участок только из-за присутствия промышленного слова.

В основную аналитическую выборку попадают только записи со статусом
`included`. Записи `manual_review` должны быть проверены специалистом до
расчёта итоговых показателей. Для аудита сохраняются нормализованная группа,
метка, статус, найденное ключевое слово и причина исключения.

## Пилотный пайплайн обработки таблицы

Пилотный пайплайн принимает локальный CSV без обращения к НСПД и выполняет
следующие шаги:

1. Проверяет наличие обязательных столбцов.
2. Преобразует `area_m2` и `cadastral_value` в числовой формат.
3. Маркирует строку как `invalid`, если площадь или кадастровая стоимость
   отсутствует, не является конечным числом либо меньше или равна нулю.
4. Рассчитывает `cadastral_value_m2` только для валидных строк.
5. Применяет словарный классификатор ВРИ ко всем строкам, включая строки с
   пустым ВРИ и невалидными числовыми полями.
6. Сохраняет полный результат в Parquet без молчаливого удаления строк.

Невалидные строки сохраняются для аудита с полями `data_quality_status` и
`data_quality_reason`, но не должны включаться в расчёт аналитических метрик.
Пример входной таблицы содержит только синтетические идентификаторы
`SYNTH-*`, не являющиеся кадастровыми номерами.

Запуск из корня проекта:

```bash
python -m siberia_price.pipeline.process_land_table \
  --input data/examples/industrial_land_sample.csv \
  --output data/processed/industrial_land_sample_processed.parquet
```

## Raw-слой и трассируемость данных

До любой нормализации каждый полученный в будущем ответ НСПД сохраняется в
raw-слое как самостоятельный JSON-файл. Raw-файл считается неизменяемым
исходным свидетельством: исправления выполняются в коде парсинга или
последующих слоях, а не редактированием исходного ответа.

Для каждой processed-записи должна сохраняться возможность определить:

- исходный raw JSON;
- версию правил парсинга и классификации ВРИ;
- причины признания строки невалидной или исключённой;
- параметры запуска, сформировавшего processed Parquet.

Воспроизводимый маршрут данных:

```text
raw JSON → parser → плоская таблица → ВРИ и проверки качества → Parquet
```

Локальные тесты используют только файлы `data/examples/nspd/`. Они не
обращаются к интернету. Сетевой клиент НСПД остаётся нереализованным, а
массовые запросы запрещены до отдельного согласования источника, лимитов и
правил хранения.
